스프링부트 프로젝트의 실행방법, 즉 프로젝트를 서버로 배포하는 방법

    src/main/java 하위 패키지에 .java로 시작하는 메인이 존재하는 파일이 존재했다. 그 파일이 프로젝트 생성 시 안 만들어도 생성되어 있다.
    그 자바 파일은 프로젝트이름Appication.java

    해당 클래스에는 main메서드가 존재 => 실행되면 내장된 톰캣 서버가 배포를 시작한다

    배포는 하지만 브라우저가 자동으로 열리지 않기에 각자 브라우저 주소에 입력해서 접속해야

    


    
프로젝트 구조

    
    
      
  • src/main/java: 자바 소스코드를 모아두는 폴더
    • 
      
      
  • src/main/resources: 자바에서 사용할 자원이나 설정 파일들을 모아두는 폴더 (아래 3개가 주요 자식)
        
      
          
    • templates: 응답 화면을 모아두는 폴더 (자바 템플릿 엔진인 타임리프를 통해 만들어지는 HTML 화면 구성)
      • 
          
    • static: templates에 적용될 정적 자원을 모아두는 폴더 (주로 JS, CSS, 이미지 파일 등)
      • 
          
    • application.properties: Spring Boot 프로젝트 전반에 대한 설정을 정의하는 파일 dependency가 여기에
      • 
        
    
      
    • 
    
      
  • Project and External Dependencies: build.gradle에 명시된 라이브러리들의 모음 (Gradle이 외부에서 자동으로 다운받은 것)
    • 
    
      
  • build.gradle: Gradle(빌드 도구)의 설정 파일로, 프로젝트에 필요한 라이브러리, 빌드 방식, 배포 설정 등을 여기에 작성함
